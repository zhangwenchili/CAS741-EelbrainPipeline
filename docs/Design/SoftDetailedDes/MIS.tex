% 4 Notation
\section{Notation}


The structure of the MIS for modules comes from \citet{HoffmanAndStrooper1995}, with the addition that template modules have been adapted from \cite{GhezziEtAl2003}.  The mathematical notation comes from Chapter 3 of \citet{HoffmanAndStrooper1995}.  For instance, the symbol := is used for a multiple assignment statement and conditional rules follow the form $(c_1 \Rightarrow r_1 | c_2 \Rightarrow r_2 | ... | c_n \Rightarrow r_n )$.


The following table summarizes the primitive data types used by \progname{}. 


\begin{center}
\renewcommand{\arraystretch}{1.2}
\noindent 
\begin{tabular}{l l p{7.5cm}} 
\toprule 
\textbf{Data Type} & \textbf{Notation} & \textbf{Description}\\ 
\midrule
    character & char & a single symbol or digit\\
    integer & $\mathbb{Z}$ & a number without a fractional component in (-$\infty$, $\infty$) \\
    natural number & $\mathbb{N}$ & a number without a fractional component in [1, $\infty$) \\
    real & $\mathbb{R}$ & any number in (-$\infty$, $\infty$)\\
\bottomrule
\end{tabular} 
\end{center}


\noindent The specification of \progname{} \ uses some derived data types: sequences, strings, and tuples. Sequences are lists filled with elements of the same data type. Strings are sequences of characters. Tuples contain a list of values, potentially of different types. In addition, \progname{} uses functions, which are defined by the data types of their inputs and outputs. Local functions are described by giving their type signature followed by their specification.

\begin{center}
\begin{tabular}{p{3cm} p{4cm} p{4cm} p{2cm}}
\hline
    \textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\
\hline
    \lstinline|_register_field| & field name, values, default value & None & \lstinline|ValueError| \\
    \lstinline|get| & field name & current field value & None \\
    \lstinline|set| & field name, value & None & \lstinline|ValueError| \\
    \lstinline|iter| & field names & iterator & None \\
    \lstinline|get_field_values| & field name & all field values & None \\
\hline
\end{tabular}
\end{center}




\subsection{Semantics}


\subsubsection{State Variables}


\begin{itemize}
    \item \lstinline|self._fields: dict| -- A dictionary of field names and their current values
    \item \lstinline|self._field_values: dict| -- A dictionary of field names and their possible values
\end{itemize}




\subsubsection{Environment Variables}


None




\subsubsection{Assumptions}


None




\subsubsection{Access Routine Semantics}


\noindent \lstinline|_register_field(field_name: string, values: sequence, default_value: string)|:
\begin{itemize}
    \item transition: add field name, values, default value to \lstinline|self._fields| and \lstinline|self._field_values|
    \item output: None
    \item exception: \lstinline|ValueError| if default value doesn't exist in values or if field name already exists
\end{itemize}


\noindent \lstinline|get(field_name: string)|:
\begin{itemize}
    \item transition: None
    \item output: returns current field value (string), recursively formats format strings using other field values if applicable
    \item exception: None
\end{itemize}


\noindent \lstinline|set(field_name: string, value: string)|:
\begin{itemize}
    \item transition: change current value of the field in \lstinline|self._fields|
    \item output: None
    \item exception: \lstinline|ValueError| if value is not in allowed values
\end{itemize}


\noindent \lstinline|iter(field_names: sequence)|:
\begin{itemize}
    \item transition: None
    \item output: Python iterator over all possible field value combinations
    \item exception: None
\end{itemize}


\noindent \lstinline|get_field_values(field_name: string)|:
\begin{itemize}
    \item transition: None
    \item output: all possible values of the field in \lstinline|self._field_values| (sequence of strings)
    \item exception: None
\end{itemize}




\subsubsection{Local Functions}


None

    \item exception: None

\begin{itemize}
    \item transition: invoke MNE-Python function to produce ICA result and save to disk as .fif file

\begin{itemize}
    \item transition: None
